# Phase 2.5: 8 项 UI/UX 优化

## 概要

根据用户反馈，对方向树探索功能进行 8 项 UI/UX 优化，分 3 轮实现。

---

## 第 1 轮（P0）：视觉和交互快速修复

### 1. 去掉"派生"文字
- **文件**: `src/canvas/edges/reference-edge.tsx`
- **改动**: `derived` 关系的 label 改为空字符串（不显示文字）
- **验证**: 连线上不再显示"派生"二字

### 2. 连线改为实线
- **文件**: `src/canvas/edges/reference-edge.tsx`
- **改动**: `derived` 关系去掉 `strokeDasharray`，改为实线
- **验证**: 所有连线都是实线

### 3. 节点宽度跟随内容
- **文件**: `src/canvas/nodes/direction-node.tsx`, `src/canvas/nodes/idea-node.tsx`
- **改动**: 去掉 `w-[420px]` 固定宽度，改为 `w-fit min-w-[200px] max-w-[500px]`
- **验证**: 短文字节点窄，长文字节点宽

### 4. 展开输入框可收缩
- **文件**: `src/canvas/nodes/direction-node.tsx`, `src/canvas/store/canvas-store.ts`
- **改动**:
  - 添加 `cancelExpanding(nodeId)` action（设 isExpanding=false, opinionDraft=''）
  - 输入框右侧加 X 关闭按钮
  - ESC 键触发收缩
  - 点击节点外部：用 `useRef` + `useEffect` 监听 mousedown，如果 opinionDraft 为空则自动收缩，有内容则不收缩
- **验证**:
  - 空输入框 + 点外部 → 收缩
  - 有内容 + 点外部 → 不收缩
  - 有内容 + ESC/X → 收缩

### 6. 去掉关键词标签，精简为文字
- **文件**: `src/canvas/nodes/direction-node.tsx`
- **改动**: 去掉 keywords 的 Badge 组件，改为标题后面加灰色小字描述（用 summary 的前30字）
- **节点样式**: `桌面软件方案  Windows/Mac原生，离线可用`
- **验证**: 不再显示 Badge 标签

---

## 第 2 轮（P1）：布局算法 + 面板重构

### 5. 节点防重叠（垂直累积方案）
- **文件**: `src/canvas/lib/tree-layout.ts`, `src/canvas/store/canvas-store.ts`
- **改动**:
  - `computeChildPositions` 增加参数 `existingNodes: Array<{x,y,width,height}>`
  - 在目标 x 列上，找到已有节点占据的最低 y 坐标
  - 新子节点从该最低点之后开始排列，避免重叠
  - 节点高度估算为 50px（紧凑模式）
  - searchIdea 和 submitOpinion 调用时传入当前 nodes 列表
- **验证**: 同一父节点多次展开子方向后，新节点不覆盖旧节点

### 7. 右侧面板改为折叠式精简布局
- **文件**: `src/components/decision-drawer.tsx`
- **改动**:
  - 三个区块（已确认/待定/下一步）改为可折叠
  - 标题行：图标 + 标题 + 数量 + 箭头（点击切换展开/折叠）
  - 折叠时只显示标题行
  - 展开时：每项精简为一行文字（不用 Card 包裹）
  - 参考用户给的第三张截图样式
- **验证**: 点击箭头可折叠/展开，面板更紧凑

---

## 第 3 轮（P2）：面板拖拽交互

### 8. 面板项可删除、拖拽排序、跨区拖拽 + 画布联动
- **依赖**: `npm install @dnd-kit/core @dnd-kit/sortable @dnd-kit/utilities`
- **文件**: `src/components/decision-drawer.tsx`, `src/canvas/store/canvas-store.ts`
- **改动**:
  - 每项后面加 X 删除按钮 → 调用 store action 把节点 status 改回 idle
  - 使用 @dnd-kit 实现同区排序拖拽
  - 使用 @dnd-kit 实现跨区拖拽（已确认 ↔ 待定 ↔ 下一步）
  - 拖拽后更新 store 中节点的 status，画布节点颜色自动联动
  - 添加 store actions: `removeFromPanel(nodeId)`, `moveToCategory(nodeId, newStatus)`
- **验证**:
  - 点 X 删除 → 面板移除 + 画布节点恢复 idle 颜色
  - 拖拽排序 → 面板内顺序改变
  - 跨区拖拽 → 节点 status 切换 + 画布颜色联动
