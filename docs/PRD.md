# PRD - Idea Brainstorming

## 一、产品定位

这是一个帮助非专业用户梳理需求、拆解问题、补齐方案盲区的 AI 头脑风暴工具。

它面向的不是“已经知道怎么做的人”，而是：

- 想做一个网站，但不知道技术选型
- 想做一个 App，但不知道要不要支持手机
- 想做一个小工具，但不知道是否涉及自动更新、数据库、迁移、权限、部署、备份
- 知道自己有一个需求，但不知道该如何系统化提问

产品的价值是：让 AI 不只是回答问题，而是主动帮助用户发现“还没想到但应该考虑”的事项。

## 二、核心目标

产品长期目标：

- 帮助用户从一个模糊想法出发
- 通过 AI 对话不断补全上下文
- 自动提出关键决策点和遗漏项
- 把推导过程沉淀成结构化节点图
- 最终生成较完整的需求草案、技术方案或执行清单

当前版本目标：

- 保留现有四周壳层 UI
- 把中间主区替换为无限画布
- 实现文本卡片、聊天卡片、引用连线
- 完成“文本 -> 对话 -> 笔记 -> 分支”闭环
- 为后续“AI 主动建议”能力打基础

## 三、用户真正想解决的问题

用户原始诉求不是单纯聊天，而是：

1. 我有一个想法，但表述不完整
2. 我不知道还需要考虑哪些维度
3. 我不会专业术语，不知道该怎么问
4. 我希望 AI 主动提醒我哪些问题没想清楚
5. 我希望这些建议有结构，不要丢在一长串聊天记录里

这意味着产品除了“画布”之外，最终还要具备“需求探索助手”的能力。

## 四、产品分层理解

### 层 1：画布工作台

负责把内容可视化组织起来：

- 文本卡片
- 对话卡片
- 引用关系
- 分支推演

### 层 2：需求探索助手

负责主动发现遗漏：

- 技术选型建议
- 终端支持建议
- 自动更新建议
- 数据库与迁移建议
- 数据安全与备份建议
- 登录、权限、审计建议
- 部署、监控、日志建议

### 结论

这两个层次都重要，但开发顺序不一样：

- 先把画布工作台做出来
- 再把“AI 主动建议”叠加进去

## 五、为什么不能一开始把所有建议功能一起做完

不是因为它们不需要，而是因为现在的主区基础设施还没稳定。

如果先同时做：

- 画布
- 节点系统
- AI streaming
- 局部引用
- 主动建议引擎
- 决策抽屉联动
- checklist 模板系统

很容易把第一版拖得过大，最后每块都不稳定。

更好的顺序是：

### Phase 1：先做基础画布闭环

先证明下面这条链路稳定：

```text
文本卡片
-> 拖线创建对话
-> AI 回复
-> 展开为笔记
-> 再次分支
```

### Phase 2：再做“AI 主动建议”

在 Chat 卡片或侧边建议面板里增加：

- 你可能还需要考虑哪些问题
- 当前需求缺少哪些关键信息
- 技术方案上的待确认事项

### Phase 3：再做结构化交付

例如输出：

- 需求文档
- 技术选型清单
- 风险列表
- 决策摘要

## 六、当前版本必须支持的画布能力

1. 无限画布 + 点阵背景
2. 缩放百分比显示
3. 文本卡片可编辑、可拖动、可调大小
4. 文本卡片最大宽度限制
5. 上传 markdown 文档并渲染
6. 拖线到空白区域自动创建 Chat 卡片
7. Chat 自动引用来源文本
8. Chat 内独立输入框 + mock 流式回复
9. “展开笔记”生成新文本卡片
10. 一个节点可以分支出多个对话
11. 局部引用创建分支对话
12. 边可点击并跳回引用来源
13. 底部浮动工具栏

## 七、后续明确要做，但不要求首版完成的能力

这些不是“不做”，而是“第二阶段再做”：

1. AI 主动给出需求探索建议
2. 自动识别项目类型
3. 针对网站 / App / 小工具给出不同问题清单
4. 技术选型提示
5. 终端平台支持建议
6. 自动更新建议
7. 数据库、迁移、回滚、备份建议
8. 权限、审计、安全建议
9. 部署和运维建议
10. 决策沉淀到右侧抽屉

## 八、产品方向上的关键判断

你原本的需求没有错。

你要的不是一个普通画布，而是一个：

**“能陪你一起补全需求和技术决策的 AI 需求探索画布。”**

所以我前面说的“不做数据库迁移、权限、自动更新”等，不应理解为产品永远不涉及这些内容。

正确理解应该是：

- 这些正是未来要提示用户考虑的关键事项
- 只是当前开发阶段先不把它们做成完整业务系统
- 第一版先把承载这些建议的画布和交互搭起来

## 九、当前技术建议

当前版本最优实现：

- React
- TypeScript
- Tailwind
- Zustand
- `@xyflow/react`

原因：

- 当前核心是业务节点图，不是自由绘图
- 自定义节点、拖线建节点、引用边语义与 React Flow 更匹配
- 后续在节点上叠加“建议模块”也更自然
