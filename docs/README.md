## 这个产品现在应该怎么理解

它有两个层次：

### 第一层：画布工作台

负责承载：

- 文本卡片
- 聊天卡片
- 分支对话
- 引用关系

### 第二层：需求探索助手

负责主动给用户建议：

- 你还需要考虑哪些问题
- 技术选型有哪些方向
- 是否需要支持手机
- 是否涉及自动更新
- 是否需要数据库迁移和保护
- 是否有权限、安全、部署等问题

## 当前开发顺序

先做第一层，再叠加第二层。

原因不是第二层不重要，而是第二层必须建立在稳定的画布交互之上。

## 当前正式主方案

- 画布使用 `@xyflow/react`
- 不再沿用旧的 suggestion nodes 主区
- 不使用 `tldraw` 作为正式主实现
- 真实 AI 是正式目标，第一版实现允许先 mock，但架构必须按真实 AI 接入设计
